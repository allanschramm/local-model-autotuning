Peças do sistema

Motor
Programa que carrega os pesos e gera tokens (ex.: llama-server). Não é o arquivo do modelo.
Modelo
Pesos no disco — tipicamente um arquivo .gguf.
Quant (quantização)
Compressão dos pesos (Q4, Q5, Q8…). Menos bits → menos VRAM; TPS costuma subir. Não é “treinar de novo”.
GGUF
Formato de arquivo de modelo para motores tipo llama.cpp.
Draft / MTP
Arquivo auxiliar de especulação (fora do escopo da Semana 1 de TPS básico).

Memória e velocidade

VRAM
Memória da GPU. Pesos + KV competem por ela. Dense deve caber na VRAM física neste curso; MoE pode offload (Dia 3).
RAM
Memória do sistema (CPU). Offload de especialistas MoE pode usar RAM — não é “free speed”.
KV / contexto
Memória do histórico da conversa. Mais -c → mais KV → mais memória.
TPS
Tokens por segundo — métrica de velocidade da Semana 1. Não mede “inteligência”.
Offload
Colocar parte do modelo fora da VRAM (CPU/RAM). Em dense: evitar. Em MoE: ferramenta do Dia 3.
OOM
Out of Memory — VRAM (ou RAM) esgotada. Em denso, corte contexto/KV, remova draft ou escolha GGUF menor; não force spill.

API e uso

API local
Servidor HTTP na sua máquina (ex.: 127.0.0.1:18080), em geral compatível com o formato OpenAI.
Harness
Cliente que fala com a API: script, app, agente ou IDE.
Amostragem
Como o motor escolhe o próximo token (temperatura, top-k…). Semana 2 — qualidade, não TPS.
MCP
Model Context Protocol — protocolo para o agente usar ferramentas externas (Semana 2).
Hook / gate
Automação ou portão que bloqueia ações perigosas / Trial inválido (Semana 2).
