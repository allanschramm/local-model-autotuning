Skills e MCPs — Expandindo o Modelo Local

Um LLM por si só apenas prevê palavras. Para se tornar um verdadeiro assistente, ele precisa de ferramentas para ler arquivos, buscar na web e executar comandos. Hoje aprendemos o padrão MCP (Model Context Protocol) e a estrutura de Skills.

← Dia 1 Guia Dia 3 →

1. O que é o MCP (Model Context Protocol)?

O MCP é um padrão aberto que conecta modelos de IA (seja no Claude, VS Code ou local) a servidores de dados e ferramentas locais de forma segura. Em vez de escrever integrações manuais para cada banco de dados ou API, o MCP padroniza a comunicação.

🔌 Como Funciona a Arquitetura MCP

[Modelo / Cliente IA] ↔ (Protocolo JSON-RPC) ↔ [Servidor MCP Local] ↔ (Sistema de Arquivos / API / Banco)

  • Tools (Ferramentas): Ações que a IA pode executar (ex: read_file, search_web, run_query).
  • Resources (Recursos): Dados estáticos ou documentos expostos para leitura da IA.
  • Prompts: Templates prontos para iniciar tarefas complexas.

2. O que são Skills (Habilidades)?

Uma Skill é um pacote de instruções estruturadas (geralmente em formato SKILL.md) que ensina a IA como desempenhar um papel específico ou seguir um fluxo detalhado (ex: fazer Code Review, diagnosticar bugs ou criar gráficos).
