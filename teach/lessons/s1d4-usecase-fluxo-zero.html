1. O Recap Consolidado dos Primeiros Dias

Antes de integrar sua aplicação, revise a fundação técnica construída no curso:

🧠 Anatomia da IA

Modelos são arquivos estáticos (.gguf). A conversa vive na Janela de Contexto e o texto é fatiado em tokens numerados.

⚡ VRAM vs RAM

Denso deve caber inteiro na VRAM física. Em MoE, N_CPU_MOE mantém especialistas na CPU/RAM.

📉 Quantização (GGUF)

Reduz a precisão (ex.: Q4_K_M) e a memória necessária, com perda mensurável que depende do modelo.

🛠️ Resolução de Erros

OOM denso se resolve reduzindo contexto/KV, removendo draft ou escolhendo GGUF menor. Não derrame camadas na RAM.

2. Manual de Implementação em 4 Passos

Passo 1

Escolher o Modelo Ideal para seu Hardware

Garanta que modelo, contexto e KV caibam na VRAM física. Use scripts/check_hardware.py como ponto de partida e confirme no model card.

Passo 2

Iniciar o Servidor de API Local

Ajuste autoresearch/core/config.py e use o helper canônico do repositório: 

.\venv\Scripts\python.exe scripts\serve-config.py print-cmd
.\venv\Scripts\python.exe scripts\serve-config.py serve
Passo 3

Conectar seu Código ao Servidor Local

O helper expõe uma API HTTP compatível em http://127.0.0.1:18080/v1/chat/completions. Qualquer script ou app pode enviar requisições JSON simples.

Passo 4

Validar Velocidade e Ajustar Desempenho

Rode .\venv\Scripts\python.exe scripts\verify_setup.py --port 18080. Se o TPS estiver baixo, reduza contexto/KV ou escolha um denso menor; não reduza camadas para usar RAM compartilhada.
