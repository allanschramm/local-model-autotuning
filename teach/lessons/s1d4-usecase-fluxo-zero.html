1. O Recap Consolidado dos Primeiros Dias

Antes de integrar sua aplicação, revise a fundação técnica construída no curso:

🧠 Anatomia da IA

Modelos são arquivos estáticos (.gguf). A conversa vive na Janela de Contexto e o texto é fatiado em tokens numerados.

⚡ VRAM vs RAM

A GPU (VRAM) dá velocidade máxima (TPS). Para modelos gigantes (MoE), o offload divide as camadas mantendo a estabilidade.

📉 Quantização (GGUF)

Reduz a precisão (ex.: Q4_K_M), economizando até 75% de memória com perda imperceptível de qualidade em relação ao modelo original.

🛠️ Resolução de Erros

OOM se resolve diminuindo o contexto (-c) ou offload (-ngl). Conflito de porta 8080 se resolve alterando para --port 8081.

2. Manual de Implementação em 4 Passos

Passo 1

Escolher o Modelo Ideal para seu Hardware

Garantir que o modelo caiba na memória dedicada da sua placa de vídeo. Para GPUs com 8 GB de VRAM, escolha modelos de 7B a 8B parâmetros quantizados em Q4_K_M (ex.: Qwen2.5-7B-Instruct-Q4_K_M.gguf).

Passo 2

Iniciar o Servidor de API Local

Execute o llama-server no terminal configurando as flags otimizadas para GPU e suporte a Flash Attention: 

.\llama.cpp\build-cuda\bin\llama-server.exe -m models/unsloth/Qwen2.5-7B-Instruct-GGUF/Qwen2.5-7B-Instruct-Q4_K_M.gguf -ngl 99 -c 8192 -fa on --host 127.0.0.1 --port 8080
Passo 3

Conectar seu Código ao Servidor Local

O servidor local expõe uma API HTTP compatível com o padrão OpenAI em http://127.0.0.1:8080/v1/chat/completions. Qualquer script ou app pode enviar requisições JSON simples.

Passo 4

Validar Velocidade e Ajustar Desempenho

Verifique o log do servidor para garantir que a geração mantém uma taxa alta de tokens por segundo (TPS). Se a velocidade cair, ajuste o contexto (-c) ou reduza as camadas na GPU.
