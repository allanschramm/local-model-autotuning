1. O Guia de Primeiros Socorros

Abaixo estão os 3 erros mais frequentes enfrentados por iniciantes ao executar modelos com llama.cpp ou APIs locais:

Erro #1: Out of Memory (OOM / Memória de Vídeo Esgotada)

Sintoma: O terminal fecha sozinho do nada ou exibe a mensagem CUDA out of memory ou failed to allocate buffer.

Causa: Modelo, contexto e KV exigem mais VRAM física do que a GPU possui.

Solução rápida:
1. Reduza o CTX_SIZE configurado, sem passar do mínimo 2048.
2. Use KV quantizado e remova draft especulativo, se estiver ativo.
3. Escolha um GGUF menor. Modelo denso deve caber inteiro na VRAM física; não reduza -ngl para derramar camadas na RAM.

Erro #2: Porta de API Ocupada (Address already in use)

Sintoma: O servidor de API local falha com a mensagem failed to bind port 18080 ou WINSOCK error.

Causa: Já existe um servidor antigo rodando em segundo plano no seu computador usando essa mesma porta de rede.

Solução rápida:
Pare a instância antiga com .\venv\Scripts\python.exe scripts\serve-config.py stop e suba novamente com serve-config.py serve. Se outro aplicativo possuir a porta, ajuste a configuração desse serviço conscientemente.

Erro #3: A IA gera caracteres estranhos ou repetições infinitas

Sintoma: A resposta vira um loop de frases idênticas ou gera símbolos como 11111111 ou códigos quebrados.

Causa: O arquivo utilizado não é uma versão de instrução (Instruct/Chat) ou o parâmetro de repetição está desligado.

Solução rápida:
Certifique-se de que o nome do arquivo GGUF termina em -Instruct (ex.: Qwen2.5-7B-Instruct-Q4_K_M.gguf) e use a flag --repeat-penalty 1.1.
